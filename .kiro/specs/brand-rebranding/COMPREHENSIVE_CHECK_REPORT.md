## ⚠️ 数据库版本问题说明

### 问题描述

用户反馈：后台首页显示 "极速CMS 3.0.3 Release 20251126"

### 原因分析

后台首页的版本显示代码：
```html
<td>极速CMS {$C[version]} <span class="layui-font-12">Release {$C[release]}</span></td>
```

这里的 `{$C[version]}` 和 `{$C[release]}` 是从**数据库**中读取的配置信息，不是从代码文件中读取的。

### 数据库存储位置

版本信息存储在数据库的 `kv` 表中：
- 表名: `{表前缀}kv` (例如: `jisu_kv` 或 `le_kv`)
- 键名: `cfg`
- 值: JSON格式的配置数组，包含 `version` 和 `release` 字段

### 为什么会显示旧版本？

用户在品牌重塑**之前**已经安装过系统，数据库中存储的配置信息仍然是旧版本：
```json
{
  "version": "3.0.3",
  "release": "20251126",
  ...
}
```

### 解决方案

用户有两个选择：

#### 方案1: 重新安装系统（推荐）

1. 备份重要数据（如果有）
2. 删除数据库或创建新数据库
3. 访问 `http://你的域名/install/` 重新安装
4. 新安装会自动使用新版本号 (1.0.0 / 20260423)

#### 方案2: 手动更新数据库

执行以下SQL语句更新数据库中的版本信息：

```sql
-- 更新版本信息
UPDATE {表前缀}kv 
SET v = REPLACE(
    REPLACE(v, '"version":"3.0.3"', '"version":"1.0.0"'),
    '"release":"20251126"', '"release":"20260423"'
)
WHERE k = 'cfg';

-- 示例（如果表前缀是 jisu_）
UPDATE jisu_kv 
SET v = REPLACE(
    REPLACE(v, '"version":"3.0.3"', '"version":"1.0.0"'),
    '"release":"20251126"', '"release":"20260423"'
)
WHERE k = 'cfg';

-- 示例（如果表前缀是 le_）
UPDATE le_kv 
SET v = REPLACE(
    REPLACE(v, '"version":"3.0.3"', '"version":"1.0.0"'),
    '"release":"20251126"', '"release":"20260423"'
)
WHERE k = 'cfg';
```

执行后刷新后台页面，版本信息将显示为 "极速CMS 1.0.0 Release 20260423"
